# l10n_din5008 — DIN 5008 Document Standard

## Overview
DIN 5008 is the German/Austrian/Swiss standard for the layout of business documents (letters, invoices, memos). The `l10n_din5008` module implements this standard in Odoo by defining a set of QWeb report templates (external layouts), paper formats, CSS styles, and wizard fields that together produce DIN-compliant printed documents. It is the foundational module for the `l10n_din5008_purchase` and `l10n_din5008_repair` extensions.

## Countries
Germany (`DE`), Switzerland (`CH`)

## Dependencies
- [account](modules/account.md)

## Key Models

### base_document_layout.py
```python
class BaseDocumentLayout(models.TransientModel):
    _inherit = 'base.document.layout'

    @api.model
    def _default_report_footer(self): ...

    @api.model
    def _default_company_details(self): ...

    report_footer = fields.Html(default=_default_report_footer)
    company_details = fields.Html(default=_default_company_details)
    street = fields.Char(related='company_id.street', readonly=True)
    street2 = fields.Char(related='company_id.street2', readonly=True)
    zip = fields.Char(related='company_id.zip', readonly=True)
    city = fields.Char(related='company_id.city', readonly=True)
    company_registry = fields.Char(related='company_id.company_registry', readonly=True)
    bank_ids = fields.One2many(related='company_id.partner_id.bank_ids', readonly=True)
    account_fiscal_country_id = fields.Many2one(related='company_id.account_fiscal_country_id', readonly=True)
    l10n_din5008_invoice_date = fields.Date(default=fields.Date.today, store=False)
    l10n_din5008_due_date = fields.Date(default=..., store=False)
    l10n_din5008_delivery_date = fields.Date(default=..., store=False)
```
Overrides two `web` methods (`_default_report_footer`, `_default_company_details`) to apply DIN 5008 formatting rules:
- VAT is **removed** from the footer (it is displayed separately in the DIN layout header area)
- Empty address lines are stripped to respect DIN 5008's strict formatting
- Adds wizard fields for invoice date, due date, and delivery date (defaults: today, today+7 days)

## Data Files

### data/report_layout.xml
Defines a `report.layout` record:
- `id`: `report_layout_din5008`
- `name`: `DIN 5008`
- `view_id`: `l10n_din5008.external_layout_din5008`

### report/din5008_base_document_layout.xml
The core layout definition — a QWeb template `external_layout_din5008` implementing the full DIN 5008 structure:

**Header (`.header .din_page`):**
- Table with company name (`report_header`) left, logo right
- Header height controlled by `din_header_spacing` (27mm default)
- Footer removed for DIN paper formats

**Body (`.invoice_note .din_page`):**
- Company address block: `name | street | street2 | zip | city | country` separated by `|`
- Invoice metadata table: Invoice No., Invoice Date, Invoice Date Due, Delivery Date, Source, Reference
- Partner address: commercial partner, invoice partner, shipping address
- VAT/Tax ID from the commercial partner
- Document title: "Invoice", "Draft Invoice", "Cancelled Invoice", "Credit Note", "Vendor Credit Note", "Vendor Bill"
- PDF width constrained to 180mm (`din_page_pdf` class)

**Footer (`.footer .din_page`):**
- Page number (right-aligned)
- Company details table: report_footer (left), VAT + HRB number (center), bank details (right)
- Bank info shows up to 2 bank accounts (bank name, IBAN, BIC)

### report/din5008_account_move_layout.xml
Extends `external_layout_din5008` for `account.move` documents:
- Injects `partner_id` address into the address block (not in the base template, only for invoices)

### report/din5008_report.xml
Extends `web.report_invoice_wizard_preview` with DIN 5008 document information block for the invoice preview wizard (shows sample invoice number, dates, reference in the wizard).

## Paper Formats
`l10n_din5008` defines two `report.paperformat` records:
- `paperformat_euro_din_a`: Type A — 27mm header spacing, 40mm bottom margin
- `paperformat_euro_din`: Type B — 45mm header spacing, 40mm bottom margin

Both use A4 format, portrait orientation, 70 DPI, and 20mm/10mm left/right margins.

## CSS
`static/src/scss/report_din5008.scss` — 154 lines of Sass:
- Font size: 9pt base
- PDF width: 180mm (constrained from standard Odoo)
- Header: company name positioned 10mm above logo
- Body address: 45mm fixed height, 85mm width
- Information block: 75mm width, 40mm min-height, 20mm left margin
- Footer: page number + company details with top border
- Subtotal/total lines: top border styling for readability
- `din_page_pdf` class: 180mm width with -1rem left margin to fit DIN A4

## Installation
Auto-installs with `account` and all its dependent modules. It sets up the `DIN 5008` report layout option in the company settings.

## Historical Notes
- **Odoo 17 → 18:** New in Odoo 18. This module did not exist in this form in Odoo 17. In Odoo 17, DIN formatting was typically handled by `l10n_de` (Germany-specific module). DIN 5008 was extracted as a standalone module to serve DE, AT, and CH companies with a unified document standard.
- The `l10n_din5008_invoice_date`, `l10n_din5008_due_date`, `l10n_din5008_delivery_date` fields on the layout wizard are `store=False` transient fields used only to populate the document preview — they are not persisted to the database.
- Switzerland uses a slightly modified version of DIN 5008 (QRR payment slip), but the core layout applies to both DE and CH contexts.
