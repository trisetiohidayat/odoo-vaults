## Overview

Adds POS order margin data to the Sales Analysis (Sale Report) spreadsheet and reporting views. Links `pos_sale` with `sale_margin` so that POS order margins appear alongside regular sale order margins in reports.

**Depends:** `pos_sale`, `sale_margin`

---

## Models

### `sale.report` (Extension)
**Inheritance:** `sale.report`

**Methods:**
- `_fill_pos_fields(additional_fields)` -> extends parent POS fields with `margin` calculation:
  ```sql
  margin = SUM(
    l.price_subtotal -
    COALESCE(l.total_cost, 0) / CASE COALESCE(pos.currency_rate, 0)
      WHEN 0 THEN 1.0 ELSE pos.currency_rate END
  )
  ```
  Applies currency rate conversion to `total_cost` before computing margin, ensuring correct margin calculation for multi-currency POS orders.

## Critical Notes

1. **Auto-install:** `auto_install=True`. Automatically activates when both `pos_sale` and `sale_margin` are installed together.

2. **Currency rate conversion:** The `total_cost` field (from `sale_margin`) is divided by the POS session's `currency_rate` before subtraction from `price_subtotal`. This ensures margin is expressed in the POS order's currency, not the cost currency.

3. **Report inclusion:** The `sale.report` model (aka. Sale Analysis) is the unified reporting model for both POS and sale orders. Adding the margin field here makes POS margins visible in the Sales Analysis spreadsheet/gantt views.

4. **sale.report architecture:** `sale.report` is a read-only PostgreSQL view generated by `sale.report`'s `_auto_init()` — the `_fill_pos_fields` method adds POS-specific columns to this view.
