# l10n_in_sale_stock — Indian Sale and Warehouse Management

## Overview
Extension bridging [[Modules/l10n_in_sale]] + [[Modules/l10n_in_stock]] + `sale_stock`. Adds the warehouse address to customer invoices created from Sale Orders — required for the dispatch details in Indian e-invoice JSON and e-waybill generation. Also carries GST treatment through to the invoice.

## Country
India

## Dependencies
- l10n_in_sale
- l10n_in_stock
- sale_stock

## Key Models

### `AccountMove` (`account.move`) — account_move.py
- `_inherit = "account.move"`
- Inherits GST treatment and fiscal position logic from `l10n_in_sale`

### `StockMove` (`stock.move`) — stock_move.py
- `_inherit = "stock.move"`
- `_l10n_in_get_product_price_unit()` — returns product's standard price in the move's UOM
- `_l10n_in_get_product_tax()` — returns sale taxes (or supplier taxes for incoming moves), flagged `is_from_order: False`

### `StockPicking` (`stock.picking`) — stock_picking.py
- `_inherit = 'stock.picking'`
- Overrides `_l10n_in_get_invoice_partner()` — returns partner from linked sale order (for EDI)
- Overrides `_l10n_in_get_fiscal_position()` — returns fiscal position from sale order

## Data Files
None (pure code module).

## Warehouse Address in E-Invoice
The `SaleOrder.warehouse_id` address is used as the `DispDtls` (Dispatch Details) in the e-invoice JSON generated by [[Modules/l10n_in_edi]]. The chain: Sale Order → Invoice → `_l10n_in_get_warehouse_address()` → dispatch details in e-invoice JSON. See [[Modules/l10n_in_edi]] `_get_l10n_in_edi_saler_buyer_party()` which calls `move._l10n_in_get_warehouse_address()`.

## Installation
`auto_install = True`. Auto-installs as part of the Indian sale-stock accounting stack.
