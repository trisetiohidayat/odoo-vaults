# Full MRP Manufacturing Flow — Complete

Complete end-to-end manufacturing flow from BoM to finished goods in stock, covering Odoo 17 source code.

**Source:** `odoo/addons/mrp/models/mrp_production.py`

## Flow Diagram

```
Bill of Materials (mrp.bom) ── components + operations defined
    ↓
Manufacturing Order Created (draft)
    ↓ action_confirm
MO Confirmed (confirmed) ── move_raw_ids, move_finished_ids created
    ↓ action_assign (or scheduler)
MO Reserved (confirmed) ── components reserved at warehouse location
    ↓
MO In Progress (progress) ── workorders executing, qty_producing entered
    ↓
MO To Close (to_close) ── all workorders done, qty complete
    ↓ button_mark_done
MO Done (done) ── finished goods received in stock.quant
    ↓
Finished product available in stock
```

## MO State Definitions

**`mrp.production.state`** (`mrp_production.py` line 136-150):
```python
state = fields.Selection([
    ('draft', 'Draft'),
    ('confirmed', 'Confirmed'),
    ('progress', 'In Progress'),
    ('to_close', 'To Close'),
    ('done', 'Done'),
    ('cancel', 'Cancelled')],
    compute='_compute_state', ...
)
```

**Computed state logic** (`_compute_state`, line 531):
```
draft       ← no moves exist yet
confirmed   ← moves exist, none in progress/done
progress    ← any workorder started OR qty_producing set OR any move picked
to_close    ← all workorders done AND qty_producing >= product_qty
done        ← all moves raw+finished in state done/cancel
cancel      ← cancelled or all finished moves cancelled
```

**`reservation_state`** (line 151):
```python
reservation_state = fields.Selection([
    ('confirmed', 'Waiting'),          # needs stock
    ('assigned', 'Ready'),             # fully reserved
    ('waiting', 'Waiting Another MO'), # waiting on other operations
])
```

## Step 1: Bill of Materials (BoM) Creation

**Model:** `mrp.bom`

**BoM Selection on MO** (`_compute_bom_id`):
```python
domain = [
    ('product_id', '=', product_id) OR ('product_tmpl_id', '=', product_id.product_tmpl_id),
    ('type', '=', 'normal'),
    ('company_id', '=', False OR company_id),
]
```

**BoM Types:**
| Type | `type` value | Behavior |
|------|-------------|----------|
| Kit / Phantom | `phantom` | `explode()` in MO replaces with sub-assembly components |
| Normal | `normal` | Standard manufacturing |
| Subcontracting | (same, uses subcontract route) | Components sent to subcontractor |

**BoM Line (`mrp.bom.line`):**
- `product_id` — component product
- `product_qty` — quantity needed per finished unit (e.g. 2kg flour per 1 cake)
- `product_uom_id` — unit of measure
- `bom_id` — parent BoM (for sub-assemblies = multi-level BoM)
- `operation_id` — which routing operation consumes this component
- `coste_share` — cost distribution for byproducts

**Consumption Flexibility** (`consumption` field on BoM):
| Value | Meaning | Behavior |
|-------|---------|----------|
| `strict` | Strict consumption | Must consume exactly BoM qty |
| `flexible` | Flexible consumption | Can consume more or less |
| `manual` | Manual consumption | User enters qty manually per move |

## Step 3: MO Confirmation

**State:** `draft` → `confirmed`

**Trigger:** `action_confirm()` (line 1416)

```python
def action_confirm(self):
    moves_ids_to_confirm = set()
    move_raws_ids_to_adjust = set()
    workorder_ids_to_confirm = set()
    for production in self:
        # 1. Handle serial number tracking UoM normalization
        if production.product_tracking == 'serial' and ...:
            production_vals = {
                'product_qty': ...,
                'product_uom_id': production.product_id.uom_id
            }
        # 2. Collect all moves and workorders to confirm
        move_raws_ids_to_adjust.update(production.move_raw_ids.ids)
        moves_ids_to_confirm.update((production.move_raw_ids | production.move_finished_ids).ids)
        workorder_ids_to_confirm.update(production.workorder_ids.ids)

    # 3. Adjust procurement method for raw moves
    move_raws_to_adjust._adjust_procure_method()

    # 4. Confirm all moves (creates stock.move records)
    moves_to_confirm._action_confirm(merge=False)

    # 5. Confirm all workorders
    workorder_to_confirm._action_confirm()

    # 6. Trigger scheduler for forecasted shortages
    self.move_raw_ids._trigger_scheduler()

    # 7. Confirm internal pickings (MO's own transfers)
    self.picking_ids.filtered(lambda p: p.state not in ['cancel', 'done']).action_confirm()

    # 8. Set state to confirmed for draft MOs
    self.filtered(lambda mo: mo.state == 'draft').state = 'confirmed'
```

**What is created during confirmation:**

**`move_raw_ids`** (components) — One `stock.move` per BoM line:
```
product_id:     component product
product_qty:    product_qty × bom_line.product_qty (UoM converted)
location_id:    production.location_src_id (warehouse stock)
location_dest_id: production.location_id (production location /女儿)
state:          'waiting' (needs reservation)
raw_material_production_id: self.id
```

**`move_finished_ids`** (finished goods) — One `stock.move`:
```
product_id:     self.product_id
product_qty:    self.product_qty
location_id:    production location (stock location)
location_dest_id: location_dest_id (stock / finished goods)
state:          'waiting'
production_id:  self.id
```

**`workorder_ids`** — One per routing operation in BoM, linked to workcenter.

## Step 4: Material Reservation

**State:** `confirmed` (reservation state may change to `assigned` or stay `confirmed`)

**Trigger:** `action_assign()` (line 1497) or MRP scheduler

```python
def action_assign(self):
    for production in self:
        production.move_raw_ids._action_assign()
```

**`_action_assign()` on stock.move:**
- Searches for `stock.quant` at `location_src_id` for each component
- Reserves exact quantity on each quant
- Updates `move.state` to `assigned` if fully reserved
- Updates `move.state` to `partially_available` or stays `confirmed` if partially/fully unavailable

**Components availability computed** (`_compute_components_availability`, line 343):
```python
if all moves have forecast_availability >= product_qty:
    state = 'available'
elif any move has forecast_availability < 0:
    state = 'unavailable'
else:
    # compute expected date vs date_start
    if expected_date > date_start:
        state = 'late'
    else:
        state = 'expected'
```

**`reservation_state` on MO** (line 460):
- `'assigned'` — all `move_raw_ids` state = `assigned`
- `'confirmed'` — any `move_raw_ids` state = `confirmed` (waiting for stock)
- `'waiting'` — any `move_raw_ids` state = `waiting` (waiting on other MO)

## Step 5: Workorder Execution

**State:** `confirmed` → `progress` (when first workorder starts)

**Trigger:** "Start" button on workorder

### Workorder State Machine

```
pending → ready → progress → done
              ↓
          pending (blocked by dependency)
```

### Workorder Methods

**`action_start()`** — Start the workorder:
- `state → 'progress'`
- `date_start = now`
- Creates `mrp.workcenter.usage` record for OEE tracking

**`action_done()`** — Mark workorder complete:
- `date_finished = now`
- `state → 'done'`
- Logs actual duration: `duration = date_finished - date_start`
- If last workorder: triggers MO state transition to `to_close`

**Time Calculation** (computed on workorder):
```python
duration_expected = workcenter_id.time_start
                  + (cycle × workcenter_id.time_cycle)
                  + workcenter_id.time_stop
```
Where `cycle = ceil(qty_producing / workcenter_id.capacity)`

### OEE (Overall Equipment Effectiveness)

```python
availability = (planned_time - downtime) / planned_time
performance  = actual_cycles / max_cycles
quality      = good_products / total_products
oee          = availability × performance × quality
```

def action_produce(self, qty_producing, mode='immediate', **kwargs):
    # 1. Set qty_producing on the MO
    self.qty_producing = qty_producing

    # 2. If serial tracking, split by serial number
    if self.product_id.tracking == 'serial':
        self._set_qty_producing()   # sets qty_producing = 1 per serial

    # 3. For each raw move: set quantity_done
    for move in self.move_raw_ids:
        move.quantity_done = qty_producing × unit_factor

    # 4. For finished move: set quantity_done
    self.move_finished_ids.filtered(...).quantity_done = qty_producing

    # 5. Create backorder if partial
    if mode == 'record_planned':
        # Just record, don't finish
        return True

    # 6. If immediate: button_mark_done()
    if mode == 'immediate':
        return self.button_mark_done()

## Step 7: MO Completion

**State:** `progress` → `to_close` → `done`

**Trigger:** `button_mark_done()` (line 2002)

```python
def button_mark_done(self):
    res = self.pre_button_mark_done()  # validate all moves have qty done
    if res is not True:
        return res

    # 1. Finish all workorders (button_finish)
    self.workorder_ids.button_finish()

    # 2. Split productions for backorders
    backorders = self._split_productions()

    # 3. Post inventory (move quants from production location to stock)
    productions_not_to_backorder._post_inventory(cancel_backorder=True)
    productions_to_backorder._post_inventory(cancel_backorder=True)

    # 4. Moves without qty done → set state='done' with product_uom_qty=0
    (productions_not_to_backorder.move_raw_ids | ...).filtered(
        lambda x: x.state not in ('done', 'cancel')).write({
            'state': 'done', 'product_uom_qty': 0.0
        })

    # 5. Write final MO state
    for production in self:
        production.write({
            'date_finished': fields.Datetime.now(),
            'state': 'done',
            'is_locked': True,
        })

    # 6. Assign any newly available backorder components
    backorders.filtered(...).action_assign()
```

## Multi-Level BoM / Phantom BoM Explosion

**`explode()` called from `_compute_move_raw_ids`**:

```python
def _compute_move_raw_ids(self):
    for production in self:
        if production.bom_id and production.product_id and production.product_qty > 0:
            # explode() expands phantom BoMs
            boms, data = production.bom_id.explode(
                production.product_id,
                production.product_qty,
                picking_type=production.picking_type_id,
            )
            moves = production._mrp_explode(
                production.product_id, production.product_qty,
                boms, data, production.location_src_id
            )
            production.move_raw_ids = moves
```

**Phantom BoM behavior (`bom_id.explode()`):**
- For each `bom_line` where line's `bom_id.type == 'phantom'`:
  - Recursively explode that sub-assembly BoM
  - Replace the kit with its actual components
- Result: flat list of all leaf components
